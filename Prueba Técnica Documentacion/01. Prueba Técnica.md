
**Requerimientos**: Es necesario desplegar una aplicación web para validación de circulación de pico y placa. 

  Se debe tener las siguientes consideraciones:

1. Tener la Consulta por número de placa y comprobar si un vehículo puede circular en una fecha o día determinado; El formulario de consulta debe permitir ingresar la placa y la fecha actual con la hora.
2. Se debe validar que la fecha ingresada no sea anterior a la fecha y hora actual.
3. Se debe mostrar el resultado con la información de placa del Vehículo y un texto que indique sí es libre de circular en la fecha y hora ingresada.
4. El aplicativo debe tener dos partes:

5. Capa de Frontend en Angular ([https://angular.io/](https://angular.io/) )
6. Capa de Backend o de lógica de negocio debe ser en Java y puede estar realizado en SpringBoot ([https://spring.io/projects/spring-boot](https://spring.io/projects/spring-boot)). 

7. La comunicación entre capas debe ser mediante Servicios REST usando HTTP.
8. Toda la lógica de consulta y validación del pico y placa debe estar en la capa de backend.
9. Se debe asumir que la aplicación debe estar lista para ser usada en ambiente productivo o de producción con las consideraciones necesarias que deben tenerse en cuenta.
10. Crear el código fuente y cargarlo en [github.com](http://github.com/) . El proyecto debe contener un archivo README indicando cómo desplegar o correr el proyecto
